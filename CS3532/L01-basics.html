Evaluating Contest Problems

  • How do we decide which problems to work on?
  • As a team, split up the problems, read them, and triage the problems
    • (A) solved before, and can solve it again quickly
    • (B) solved before, and can solve it again slowly
    • (C) seen before, and believe can solve it
    • (D) seen before, and unsure how to solve it
    • (E) never seen this before

Evaluating Contest Problems

  • Learn to reverse engineer input sizes!
  • Assume that a modern computer can do $\sim 10^8$ operations in one second (the time limit for a problem in many contests)

What Does Your Input Size Say About You?

$n \le 100M$ $O(n)$ or better Input sizes rarely exceed $n = 1M$ (I/O)
$n \le 10M$ $O(n \lg \lg n)$ i.e., Sieve of Eratosthenes
$n \le 4.5M$ $O(n \lg n)$ i.e., Merge Sort, Quick Sort
$n \le 10K$ $O(n^2)$ i.e., Bubble/Selection/Insertion Sort
$n \le 2.5K$ $O(n^2 \lg n)$ i.e., loop + Binary Search Tree
$n \le 450$ $O(n^3)$ i.e., Floyd-Warshall (all pairs shortest path)
$n \le 100$ $O(n^4)$ i.e., 3d dynamic programming + loop
$n \le 26$ $O(2^n)$ i.e., all subsets with $O(1)$ work per
$n \le 10$ $O(n!)$, $O(n^6)$ i.e., enumerating permutations
  • Given a set $S$ of $n$ points scattered on the 2d plane ($2 \le n \le 5~000$), find two points $x,y \in S$ with the greatest Euclidean distance.
    • What if $n \le 200K$? You may assume that the points are randomly scattered on the 2d plane. We can build a convex hull in $O(n \lg n)$.

Learn to Use Your Language Features

  • Efficient input handling matters
    • Even in C++, scanf() is usually faster than std::cin
    • In Java, BufferedReader.readLine() can sometimes be faster than Scanner.nextLine()
  • Learn to use what is available. For instance, did you know that Java provides a BigInteger class via java.math.BigInteger?
