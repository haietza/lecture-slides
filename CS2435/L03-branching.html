Branching / Decision Structures

Decision Structures

  • We want programs to dynamically respond to different inputs
  • One part of that is giving programs the ability to decide what portions of code to run based on certain decisions
  • One tool for creating dynamic programs is the if statement

if Statements

  • One way to think of an if statement is to consider the flowchart for the following structure:
    • 							
								if it is cold outside
									put on a jacket
  • In this example, we only put on a jacket if the condition that it is cold outside is true

if Statements

  • In Python, this would be an if statement
    • 							
								if coldOutside:
									wearJacket()
  • An if statement has two main parts:
    • The header uses the if keyword, followed by a boolean condition (or "predicate")
    • The body is the indented block of code that runs if the condition evaluates to true

Boolean Values

  • A boolean value is either true or false, represented in Python as True and False respectively
  • We infrequently type out boolean literals directly — instead, we have operators that create booleans for us
						
							x = True
							if x:
								print("Hello World")

Relational Operators

  • Relational operators create boolean values by comparing two values against one another
						
							x = float(input("x = "))   # get a value for x from the user
							y = float(input("y = "))   # get a value for y from the user
							print('eq', x == y)        # True if x equal to y
							print('ne', x != y)        # True if x not equal to y
							print('lt', x <  y)        # True if x less than y
							print('le', x <= y)        # True if x less than or equal to y
							print('gt', x >  y)        # True if x greater than y
							print('ge', x >= y)        # True if x greater than or equal to y

Relational Operators

  • Please note that assignment (=) and equality (==) are two different operators, and must not be confused with one another
  • We can use relational operators to compare any numeric values, as well as strings

Relational Operators

  • Using relational operators to compare strings is based on their lexicographic ordering
						
							print("abc" < "def")

This will print True, "abc" comes before "def"

						
							print("bed" > "bet")

This will print False, "bed" does not come after "bet"

Relational Operators

  • This can be tricky if you aren't used to how lexicographic ordering works
    • 							
								print("USD" > "euro") # False, surprisingly, but why?
							
						
    							
								print(ord("U"))   # 85
								print(ord("e"))   # 101, "U" comes before "e" in the ASCII table
  • Lexicographic ordering is not alphabetic ordering
    • 							
								print("USD".lower() > "euro".lower())  # True, that's better!

							x = 10

								print("Neither will this")

if Statements

  • if Statements can be nested, but we have to make sure we are careful with how this works!
						
							x = 10
							if A:
							    x *= 2       # this line runs if A is true
							    if B:
							        x += 5   # this line runs if both A and B are true
							print(x)

The and Operator

  • The logical and operator takes two operands that must both be boolean expressions
  • The resulting combined expression is true if and only if both operands are true
						
							A        B        A and B
							T        T        T

							F        T        F

The not Operator

  • The logical not operator is for unary negation — it inverts the truth value of its expression
						
							A        not A
							T        F
							F        T

The else Clause

  • Sometimes we want to do some behavior when our if condition is not true — One way we can accomplish this is by negating the condition
						
							if a >= 10 and a <= 50:
								print("a is between 10 and 50")
						
					
						
							if a < 10 or a > 50:
								print("a is not between 10 and 50")

This transformation is an example of DeMorgan's Law

The else Clause

  • Another way to do this is using the else clause
  • The body of the if statement runs if the condition is true, the body of the else clause runs if the condition is false
						
							if a >= 10 and a <= 50:
								print("a is between 10 and 50")
							else:
								print("a is not between 10 and 50")

The else Clause

						
							if a >= 10 and a <= 50:
								print("a is between 10 and 50")
							else:
								print("a is not between 10 and 50")
  • A maximum of one of these blocks can run — one block running will prevent the other one from being entered, even if the condition changes!

The else Clause

						
							if a >= 10 and a <= 50:
								print("a is between 10 and 50")
								a = 0 # this will not make us enter the else!
							else:
								print("a is not between 10 and 50")
  • A maximum of one of these blocks can run — one block running will prevent the other one from being entered, even if the condition changes!

The elif Clause

  • Sometimes we want to check more than one condition to determine what action to take
    • 							
								weather = input("What is the weather?")
								if weather == "rain":
									print("take an umbrella")
								elif weather == "snow":
									print("wear a jacket")
								elif weather == "sunny":
									print("wear your sunscreen")
  • Same rules apply: can only enter one block

The elif Clause

  • if-elif chains can end with else blocks, but do not have to — like a catch-all
    • 							
								weather = input("What is the weather?")
								if weather == "rain":
									print("take an umbrella")
								elif weather == "snow":
									print("wear a jacket")
								elif weather == "sunny":
									print("wear your sunscreen")
								else:
									print("enjoy your day!")

The elif Clause

  • The only way to reach the next elif clause is for all the above if and elif clauses to be false
    • 							
								if a > 10:
									print("a is greater than 10")
								elif a < 10:
									print("a is less than 10")
								elif b > 0:
									print("b is greater than zero")

Nested Structures

  • Be careful! Clauses are associated with one another based on indentation level
    • 							
								if a > 10:
									if b > 0:
										print("b is greater than zero")
									else:
										print("mystery") # when does this print?

"mystery" prints when a is greater than 10, but b is not greater than zero

Building Randomness

  • Sometimes programs use random numbers to vary behavior
    • Games need variance so behavior isn't consistent
    • Simulations can sometimes use randomness to model real-world behaviors
    • We can even approximate the solutions to math problems with random numbers

Building Randomness

  • We can import the random library to generate random values
    • 							
								import random
  • Include this import at the top of any program using the random library. Import statements almost always go at the top of the file

Building Randomness

  • The randint(a, b) function generates an integer value in the range of a to b
						
							x = random.randint(1, 5)  # can randomly generate 1, 2, 3, 4, 5
							print(2 * x + 1)          # we can use it like any other number

Building Randomness

  • The uniform(a, b) function generates a float in the range of a to b
						
							x = random.uniform(1, 5)  # any float in the range 1 to 5
							print(x)

Building Randomness

  • One simple way to create more complex random behavior is combining if statements with randint(...)
						
							x = random.randint(1, 3)  # can randomly generate 1, 2, 3
							if x == 1:
								print("behavior 1")
							elif x == 2:
								print("behavior 2")
							else:
								print("behavior 3")
