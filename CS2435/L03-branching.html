Branching / Decision Structures

Decision Structures

  • We want programs to dynamically respond to different inputs
  • One part of that is giving programs the ability to decide what portions of code to run based on certain decisions
  • One tool for creating dynamic programs is the if statement

if Statements

  • One way to think of an if statement is to consider the flowchart for the following structure:
    • 							
								if it is cold outside
									put on a jacket
  • In this example, we only put on a jacket if the condition that it is cold outside is true

if Statements

  • In Python, this would be an if statement
    • 							
								if coldOutside:
									wearJacket()
  • An if statement has two main parts:
    • The header uses the if keyword, followed by a boolean condition (or "predicate")
    • The body is the indented block of code that runs if the condition evaluates to true

Boolean Values

  • A boolean value is either true or false, represented in Python as True and False respectively
  • We infrequently type out boolean literals directly — instead, we have operators that create booleans for us
						
							x = True
							if x:
								print("Hello World")

Relational Operators

  • Relational operators create boolean values by comparing two values against one another
						
							x = float(input("x = "))   # get a value for x from the user
							y = float(input("y = "))   # get a value for y from the user
							print('eq', x == y)        # True if x equal to y
							print('ne', x != y)        # True if x not equal to y
							print('lt', x <  y)        # True if x less than y
							print('le', x <= y)        # True if x less than or equal to y
							print('gt', x >  y)        # True if x greater than y
							print('ge', x >= y)        # True if x greater than or equal to y

Relational Operators

  • Please note that assignment (=) and equality (==) are two different operators, and must not be confused with one another
  • We can use relational operators to compare any numeric values, as well as strings

Relational Operators

  • Using relational operators to compare strings is based on their lexicographic ordering
						
							print("abc" < "def")

This will print True, "abc" comes before "def"

						
							print("bed" > "bet")

This will print False, "bed" does not come after "bet"

Relational Operators

  • This can be tricky if you aren't used to how lexicographic ordering works
    • 							
								print("USD" > "euro") # False, surprisingly, but why?
							
						
    							
								print(ord("U"))   # 85
								print(ord("e"))   # 101, "U" comes before "e" in the ASCII table
  • Lexicographic ordering is not alphabetic ordering
    • 							
								print("USD".lower() > "euro".lower())  # True, that's better!

							x = int(input("x = "))
							y = int(input("y = "))
							if x > y:
								print("x is greater than y")
							if x == y:
								print("x is equal to y")
							if x != y:
								print("x is not equal to y")

if Statements

  • The body of an if statement ends when we dedent the lines back to their previous level
						
							x = 10
							if False:
								print("The condition is false, so this won't get printed")
								print("Neither will this")
								print("Nor will the next line that changes the value of x")
								x = 12
							print("This line will be printed, however")
							print("And the value of x is unchanged:", x)

if Statements

  • if Statements can be nested, but we have to make sure we are careful with how this works!
						
							x = 10
							if A:
							    x *= 2       # this line runs if A is true
							    if B:
							        x += 5   # this line runs if both A and B are true
							print(x)

Boolean Logic

  • Sometimes we need to build more complex boolean expressions by combining conditions
						
							if it is cold
							    wear a jacket
							if it is raining
							    carry an umbrella
							if it is cold and it is raining
							    wear a raincoat

Boolean Logic

  • In Python, we use the and, or, and not operators to help us express complex conditions
  • and and or take two conditions and combine them into one
								
									A        B        A and B
									-------------------------
									T        T        T
									T        F        F
									F        T        F
									F        F        F
								
							
								
									A        B        A or B
									-------------------------
									T        T        T
									T        F        T
									F        T        T
									F        F        F

The and Operator

  • The logical and operator takes two operands that must both be boolean expressions
  • The resulting combined expression is true if and only if both operands are true
						
							A        B        A and B
							T        T        T
							T        F        F
							F        T        F
							F        F        F

The or Operator

  • The logical or operator takes two operands that must both be boolean expressions
  • The resulting combined expression is true if either operand is true
						
							A        B        A or B
							T        T        T
							T        F        T
							F        T        T
							F        F        F

The not Operator

  • The logical not operator is for unary negation — it inverts the truth value of its expression
						
							A        not A
							T        F
							F        T
						
					
						
							if not (x > 10):
								print("for what values of x will this statement print?")
