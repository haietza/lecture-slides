Sets and Dictionaries

Sets

  • Used for homogeneous data
  • Key differences from lists:
    • Hold only hashable (immutable) data
    • No guaranteed order
    • No duplicates allowed

Why Use Sets?

  • Incredibly efficient for checking set membership (using in keyword)
  • Lists check linearly $O(n)$
  • Sets use hash tables for constant time $O(1)$ checks

Basic Operations

						
							x = set()      # create an empty set
							x = {1, 2, 3}  # curly braces denote a set
							print(len(x))  # result: 5, number of elements in set
							print(3 in x)  # result: True
							x.add(6)       # result: False
							x.remove(3)    # remove 3 from the set

Connectives (Union)

Elements in either set (like OR)

						
							s1 = {"a", "b", "c"}
							s2 = {"c", "d", "e"}
							result = s1.union(s2) # or s1 | s2
							print(result)         # {'a', 'b', 'c', 'd', 'e'}

Connectives (Intersection)

Elements in both sets (like AND)

						
							s1 = {"a", "b", "c"}
							s2 = {"c", "d", "e"}
							result = s1.intersection(s2) # or s1 & s2
							print(result)                # {'c'}

Connectives (Difference)

  • Elements in one set but not the other
  • Non-symmetric: $A-B \ne B-A$
						
							s1 = {"a", "b", "c"}
							s2 = {"c", "d", "e"}
							result = s1.difference(s2) # or s1 - s2
							print(result)              # {'a', 'b'}
							print(s2 - s1)             # {'d', 'e'}

Connectives (Symmetric Difference)

Elements in either set, but not both

						
							s1 = {"a", "b", "c"}
							s2 = {"c", "d", "e"}
							result = s1.symmetric_difference(s2) # or s1 ^ s2
							print(result)                        # {'a', 'b', 'd', 'e'}

Set Comprehension

  • Build a set by describing its construction
  • Syntax: {expression for element in sequence if condition}
  • Uses curly braces, similar to list comprehension

Dictionaries

  • Store values in key-value pairs
  • Use a key to retrieve/modify the associated value
  • Fast for finding, updating, removing values (like sets)
  • No guaranteed order on keys

Basic Operations

						
							x = dict()            # or x = {}
							x = {'a': 1, 'b': 2}
							print(x['b'])         # prints 2
							print(x['d'])         # KeyError
							print(x.get('b'))     # prints 2
							print(x.get('d', -1)) # prints -1, get() takes a default value

More Operations

						
							x = {'a': 1, 'b': 2}
							print('a' in x) # prints True (checks keys)
							print(len(x))   # prints 2
							x['c'] = 3      # add entry
							del x['b']      # delete pair
							print(x)        # {'a': 1, 'c': 3}

Iteration

  • Multiple ways to iterate: Keys, values, or items (key-value pairs)
  • items() method is most common
						
							d = {'a': 1, 'b': 2}
							for k in d.keys(): print(k)
							for v in d.values(): print(v)
							for k,v in d.items(): print(k, '-->', v)

Converting Data Structures

Easy conversion between simple sequence types (lists, sets, tuples)

						
							x = [1, 2, 3, 3, 4]
							s = set(x)         # {1, 2, 3, 4}
							t = tuple(x)       # (1, 2, 3, 3, 4)
							l_from_s = list(s) # [1, 2, 3, 4] (order arbitrary)

Converting Data Structures

From a dictionary: Depends on what you want to retrieve

						
							d = {'a': 1, 'b': 2}
							print(list(d))          # ['a', 'b'] (keys)
							print(list(d.keys()))   # ['a', 'b']
							print(list(d.values())) # [1, 2]
							print(list(d.items()))  # [('a', 1), ('b', 2)]

Converting Data Structures

To a dictionary: Sequence must be tuples (or sequences) of length 2 (key/value pairs)

						
							d = dict([('a', 1), ('b', 2)]) # Correct
							dict([1, 2, 3])                # Error
							dict([(1,2,3), (4,5,6)])       # Error
