Final Projects

Project Structure

  • Groups will consist of anywhere from 1-3 students (preferably 2 or 3)
  • Projects will be decided by group members and proposed via a written proposal
  • Group projects should be substantial enough to provide enough tasks to all members of the group
  • Group projects should be able to be subdivided in such a way that each student can indicate which part of the project they were responsible for

Project Topic

  • Good projects will
    • Demonstrate an understanding of data science (or at least substantial Python proficiency)
    • Be challenging enough to warrant a substantial time investment
    • Require you to learn new skills (new libraries, GUI building, new techniques, etc)
    • Draw from topics you already know and are familiar with, so you're excited about the project

Example Projects

  • Expand one of the labs into a full, robust project
    • A playable Minesweeper game
    • A playable Chess game (maybe with a weak AI opponent)
  • Many games make for great projects
    • crossword puzzle generator, sudoku solver, etc
    • Wordle strategy analysis
  • A physics simulation using PyGame
  • Data analysis using matplotlib
  • A graphical user interface (GUI) frontend to scientific calculators or tools

Project Assessment

Project Evaluation (60%)

  • Performance - does your program perform correctly as expected?
  • Organization - is your program designed well? Does it use functions/classes?
  • Documentation - does your program use comments to make it easy to read and understand?
  • Efficiency - does your program run in a reasonable amount of time for the task?
  • Teamwork - for group projects, did you contribute your portion adequately?

Presentation (25%)

  • 10-15 minute presentation
  • Each team member will speak about the part of the project they contributed
  • During final exam period

Proposal (15%)

  • Technical document to explain the plan for the project
  • Graded on a combination of technical writing skills and concept
